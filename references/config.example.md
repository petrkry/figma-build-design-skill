# Config (template)

User-specific data for this skill instance. The live copy lives in `references/config.md`,
which the agent creates from this template during setup. This template is never filled in directly.

> **Do not edit this file.** To fill the config by hand, copy it first —
> `cp references/config.example.md references/config.md` — then edit `config.md`.
> This template is git-tracked: edits here leak your DS keys into version control and are
> overwritten on `git pull`.

**Setup copies this file to `references/config.md`, then fills the live copy.**
**`references/config.md` is git-ignored — keep your DS keys out of version control.**

The placeholder entry under **Library Collections** below is the setup sentinel: while the live
`config.md` still lists it (or `config.md` is missing), the skill runs setup instead of a build.

## DS Links

**This is the section you fill in by hand** — the navigation map of your design system, and the
biggest lever on build quality. The fuller it is, the more accurately the skill goes straight to
the right component instead of searching blindly.

Replace `www.link.com` with real URLs from your DS, and rename the placeholder entries to the
actual assets in your system.

The sections below are a starting taxonomy, not a fixed schema — rename them, drop the ones you
don't need, and add your own (e.g. `Cards`, `Navigation`, `Data display`, `Illustrations`). The
skill reads whatever sections it finds under **DS Links** and matches them to a task by intent,
so custom sections work without touching any other file.
